# 小小故事家

## 通义千问 · 手机端创意 AI 参赛创意文档

---

### 一、项目概述

**小小故事家** 是一款面向小朋友的 **AI 绘本创作 App**，运行在 Android 手机上。小朋友可以用 **语音或文字** 说出/输入自己喜欢的故事想法，AI 帮他们把想象变成 **专属绘本** 或 **动漫短片**——不是看别人的故事，而是 **创造自己喜欢的故事**。

**产品定位：** 让每个孩子都能当「小小作家」，把脑海里的冒险、动物朋友、魔法世界，变成可以翻页阅读的绘本，或可以播放的动漫。

**我们的四口之家：**

| 角色 | 分工 |
|------|------|
| **爸爸** | 程序员，负责 App 开发与 AI 技术实现 |
| **妈妈** | 老师，把关故事内容是否适合儿童、语言是否易懂 |
| **哥哥右右** | 小小创作者，抢先试用、提需求、创作自己的故事和动漫 |
| **妹妹绵绵** | 小小读者，听右右（或自己）做的绘本，睡前翻页阅读 |

这是一个 **真实家庭共创** 的项目：爸爸写代码、妈妈审内容、右右创造、绵绵阅读，四个角色对应「开发—教育—创作—消费」完整闭环，也在我们自己家里完成了产品验证。

---

### 二、应用场景

**小朋友自主创作绘本** — 用语音慢慢讲出故事想法（不会打字也没关系），或文字输入；AI 生成完整故事和插画，输出可翻页的 PDF 绘本。典型用户：3–8 岁小朋友。

**把想象变成动漫** — 同一个创意可切换「动漫模式」，生成分镜剧本、角色立绘和 15–60 秒动漫短视频，适合喜欢动画的大一点的孩子。典型用户：右右这样的 6–10 岁小创作者。

**兄妹共读、家庭分享** — 右右创作，绵绵阅读；睡前一起翻绘本，或一起看刚做好的动漫，成为家庭里的专属内容。

**语音 + 文字双入口** — 绵绵年纪小用语音，右右可以用文字精细修改；满足不同年龄的表达习惯。

**妈妈课堂延伸** — 妈妈作为老师，可按教学主题（分享、勇气、友谊）快速生成定制绘本，用于家庭阅读或班级素材参考。

**成长记录** — 每次创作的故事、插画、PDF 保存在手机本地，积累成孩子的成长故事库。

**典型使用路径：**

**绘本模式：**
1. 小朋友语音或文字输入创意 → 本地语音识别（可选）+ 通义千问整理
2. AI 生成 3–30 页故事 → 小朋友/家长可编辑文字
3. 选择喜欢的画风 → 通义万象生成每页插画
4. 手机内翻页预览 → 导出 PDF → 绵绵睡前阅读

**动漫模式：**
1. 输入创意 → AI 生成分镜剧本与角色设定
2. 生成角色参考立绘 → 逐段生成动漫视频
3. 本地预览播放 → 分享给家人观看

### 三、用户痛点

**1. 小朋友想讲故事，但不会写字**

孩子想象力最丰富，但打字难、语句断续。市面 App 多面向成人，依赖键盘输入，小朋友的真实表达——「嗯…然后小兔子…又去找…」——接不住。

**2. 外面的绘本和动画，不是「我的故事」**

现成内容千篇一律，孩子希望主角叫自己的名字、情节按自己的想法走。缺少「这是右右/绵绵自己做的」的成就感和专属感。

**3. 做一本绘本太复杂**

传统流程要写作文、画插图、排版、导出，家长和孩子都没时间也没技能，好想法往往停留在口头。

**4. 创作和阅读是割裂的**

很多工具只能生成文字或单张图，做不出完整绘本；更做不出能播放的动漫。小朋友要的是 **完整作品**，不是零散素材。

**5. 手机上用起来不顺**

Web 工具在手机上难操作，难以完成「说话 → 创作 → 看绘本/看动漫 → 保存分享」一条龙。

**6. 家长担心内容安全和隐私**

儿童 AI 内容需要适龄、正向；家庭故事不希望被上传到不明服务器。需要 **本地存储 + 儿童向内容约束**。

### 四、技术方案

#### 4.1 整体架构

端侧推理层：MNN + Sherpa 流式 ASR（INT8 量化）—— 支持小朋友 **语音输入**

云端推理层：通义千问 DashScope（qwen-plus）、通义万象（wan2.6-t2i）；可选豆包 Seedream；动漫模式 Seedance 2.0

本地存储层：AsyncStorage + 文件系统（绘本 / 图片 / PDF / 动漫视频）

输出形态：**绘本（PDF 翻页）** + **动漫（分镜短视频）**

设计原则：重计算上云、轻推理下沉、作品留本地、零服务器部署。App 直连百炼 API，Key 仅存本机。

#### 4.2 模型选型

| 能力 | 模型 | 部署位置 | 选型理由 |
|------|------|----------|----------|
| 故事/分镜/翻译/提示词 | 通义千问 qwen-plus | 云端（百炼 DashScope） | 中文儿童语境理解好；支持 JSON 结构化输出；OpenAI 兼容接口便于手机集成 |
| 语音创意整理 | qwen-plus（低温度） | 云端 | 把小朋友断续口语整理成通顺故事创意，保留原意 |
| 绘本插画 | 通义万象 wan2.6-t2i | 云端 | 3:4 竖版适配绘本；与千问同一百炼生态 |
| 插画加速（可选） | 豆包 Seedream 4.5 | 云端 | 可选更快生图 |
| 动漫视频 | Doubao-Seedance-2.0 | 云端 | 15 秒/段、角色参考图驱动，输出小朋友喜欢的动漫 |
| 语音识别 | Sherpa-MNN Zipformer 中英双语（INT8） | 端侧（APK 内置） | 本地实时听写，适合不会打字的小朋友 |

#### 4.3 推理框架

**输入：语音 + 文字双通道**

- **文字输入**：直接作为故事创意，适合会打字的小朋友（如右右）
- **语音输入**：麦克风 → 端侧 MNN ASR 实时转写 → 云端 qwen-plus 整理口语 → 通顺故事创意 → 适合低龄小朋友（如绵绵）

**输出：绘本 + 动漫双形态**

- **绘本链路**：qwen-plus 生成故事 → 批量生成 image prompt → wan2.6 文生图 → 本地预览 → expo-print 导出 PDF
- **动漫链路**：qwen-plus 生成分镜 JSON → 角色立绘 → Seedance 参考图驱动视频 → 本地缓存播放

**云端推理（通义千问为核心）**

- 文本：DashScope OpenAI 兼容模式，用于故事、分镜、翻译、提示词、内容安全改写
- 图像：DashScope 多模态生成 API，模型 wan2.6-t2i

**端侧推理（MNN）**

- 框架：MNN + Sherpa-MNN JNI，Kotlin Native Module 桥接 React Native
- 模型：sherpa-mnn-streaming-zipformer-bilingual-zh-en（INT8 量化），打包进 APK

#### 4.4 端侧适配思路

**1. 为小朋友设计的交互** — 大按钮、语音「开始说话 / 结束并整理」、加载动画；支持语音和文字两种输入方式。

**2. 无后端、作品在本地** — 绘本书架、插画、PDF、动漫视频均存手机，API Key 仅存本机 AsyncStorage。

**3. 儿童内容专项** — 故事语言适合 3–8 岁；角色外貌库保证全书一致；绿网拦截后自动改写 prompt 重试；妈妈（老师）参与内容标准制定。

**4. 绘本 + 动漫一键切换** — 同一 App 内两种输出形态，满足「安静读绘本」和「看动漫」两种喜好。

**5. 渐进式生成与弱网容错** — 插画逐页生成、失败可单页重试；历史记录本地索引，随时回看。

### 五、创新点

**1. 真正面向小朋友的 AI 绘本，而非成人创作工具**

从交互到 Prompt 都按儿童设计：语音优先、语言简单、内容温馨，让 **不会打字的小朋友也能创造故事**。

**2. 语音 + 文字双输入，兄弟姐妹各取所需**

绵绵用语音讲想法，右右用文字精修；同一 App 服务不同年龄段的创作习惯。

**3. 绘本 + 动漫双输出，一份创意两种作品**

小朋友可以选择做成翻页绘本给绵绵睡前读，或做成动漫短片自己观看分享—— **创造自己喜欢的故事，而不只是消费内容**。

**4. 「端侧听 + 云端想」**

MNN 本地语音识别保护隐私、降低延迟；通义千问理解并整理儿童口语——不是冷冰冰的转写，而是懂孩子在说什么。

**5. 四口之家真实验证的产品闭环**

爸爸开发、妈妈审教、右右创作试用、绵绵阅读反馈—— **家庭即实验室**，需求真实、迭代快速、情感温度天然存在。

**6. 零服务器、隐私友好**

无需自建后端；创作历史与作品本地保存，适合家庭场景对儿童数据的敏感需求。

**7. 通义生态深度整合**

qwen-plus 负责「听懂和写好故事」，wan2.6 负责「画好绘本」；主链路完整覆盖小朋友 AI 绘本的核心能力。

### 七、总结

StoryCraft 是一款 **小朋友的 AI 绘本 App**：支持 **语音和文字** 输入，输出 **绘本和动漫**，让每个孩子把想象变成属于自己的作品。

在我们家，爸爸写代码、妈妈把内容关、右右创造试用、绵绵开心阅读—— technology 服务于 family，AI 服务于孩子的想象力。

StoryCraft 将 **通义千问** 与 **MNN 端侧语音** 结合，在手机上实现：「 **小朋友说得出来，AI 画得出来，右右做得出来，绵绵读得开心** 」。
